# 开发任务 SPEC 模板

复制本模板给开发执行 agent 使用。填写时尽量具体，避免让 agent 自行扩大范围。

## 任务目标

一句话说明要完成什么。

## 背景

说明为什么要做这个任务，以及相关上下文。

## 范围内

- 明确允许修改的功能、路径或行为。
- 明确需要同步更新的前后端接口、类型或文档。

## 范围外

- 明确不处理的已知问题。
- 明确不允许做的重构、视觉调整、依赖升级或行为变化。

## 相关路径

- `apps/api/...`
- `apps/web/...`
- `docs/...`

## 行为要求

- 用户可观察到的行为应该如何变化。
- 错误态、空状态、加载态或边界情况如何处理。
- 如果发现需要扩大范围，先停下来报告，不要擅自继续。

## 验证命令

```bash
# 只保留本任务真正需要的命令
pnpm test:web
pnpm build:web
pnpm test:api
```

## 已知问题

- 写明哪些失败是既有问题，不属于本任务。
- 写明哪些命令预期可能失败，以及失败时应该如何报告。

## 交付要求

- 总结修改了什么。
- 列出运行过的验证命令和结果。
- 如果没有运行某个重要验证，说明原因。
- 默认不 commit、不 push，除非用户明确要求。
